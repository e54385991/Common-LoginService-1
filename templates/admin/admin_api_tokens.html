API接口文档
权限说明：
  • balance - 余额管理：增加/减少用户余额
  • vip - VIP管理：设置VIP等级和到期时间
  • user - 用户创建：创建新用户账号（可设置VIP、余额、强制UID）
  • password - 密码管理：修改用户密码
  • all - 所有权限：包含以上全部权限
1. 更新用户余额 需要 balance 权限

增加或减少指定用户的余额，适用于积分兑换、充值、消费扣款等场景。

POST /api/external/balance
Content-Type: application/json
X-API-Key: your-token

{
  "user_id": 1,
  "amount": 100.00,  // 正数增加，负数减少（不能使余额变为负数）
  "reason": "积分兑换充值"  // 可选，变更原因（会记录在余额日志中）
}

成功响应 (200)：

{
  "success": true,
  "message": "余额更新成功",
  "data": {
    "user_id": 1,
    "balance": 200.00
  }
}

余额不足错误 (400)：

{
  "success": false,
  "message": "余额不足，无法执行此操作",
  "data": {
    "current_balance": 50.00,
    "requested_amount": -100.00
  }
}
2. 设置用户VIP等级 需要 vip 权限

设置用户的VIP等级，适用于VIP购买、升级、降级等场景。

POST /api/external/vip-level
Content-Type: application/json
X-API-Key: your-token

{
  "user_id": 1,
  "vip_level": 3  // VIP等级，0表示普通用户
}

成功响应 (200)：

{
  "success": true,
  "message": "VIP等级设置成功",
  "data": {
    "user_id": 1,
    "vip_level": 3
  }
}
3. 设置VIP到期时间 需要 vip 权限

设置用户VIP的到期时间，留空表示永久VIP。

POST /api/external/vip-expire
Content-Type: application/json
X-API-Key: your-token

{
  "user_id": 1,
  "vip_expire_at": "2025-12-31T23:59:59Z"  // ISO 8601格式，留空表示永久VIP
}

成功响应 (200)：

{
  "success": true,
  "message": "VIP到期时间设置成功",
  "data": {
    "user_id": 1,
    "vip_level": 3,
    "vip_expire_at": "2025-12-31T23:59:59Z"
  }
}
4. 查询用户信息 无需特定权限

查询指定用户的详细信息。

GET /api/external/user?user_id=1
X-API-Key: your-token

成功响应 (200)：

{
  "success": true,
  "data": {
    "id": 1,
    "username": "testuser",
    "email": "test@example.com",
    "balance": 200.00,
    "vip_level": 3,
    "vip_expire_at": "2025-12-31T23:59:59Z",
    "is_active": true,
    "created_at": "2024-01-01T00:00:00Z"
  }
}
5. 创建用户 需要 user 权限

创建新用户账号，可同时设置VIP等级、VIP到期时间、初始余额，并支持强制指定用户ID。适用于从其他系统迁移用户、批量创建账号等场景。

POST /api/external/user
Content-Type: application/json
X-API-Key: your-token

{
  "id": 100,                              // 可选：强制指定用户ID（UID）
  "email": "user@example.com",            // 必填：邮箱地址
  "username": "newuser",                  // 必填：用户名（3-30字符，字母数字下划线）
  "password": "password123",              // 必填：密码（至少6个字符）
  "display_name": "New User",             // 可选：显示名称，留空则使用用户名
  "balance": 100.00,                      // 可选：初始余额，默认0
  "vip_level": 1,                         // 可选：VIP等级，默认0（普通用户）
  "vip_expire_at": "2025-12-31T23:59:59Z",// 可选：VIP到期时间，留空则永久
  "is_active": true                       // 可选：账号状态，默认true
}

成功响应 (200)：

{
  "success": true,
  "message": "用户创建成功",
  "data": {
    "id": 100,
    "username": "newuser",
    "email": "user@example.com",
    "display_name": "New User",
    "balance": 100.00,
    "vip_level": 1,
    "vip_expire_at": "2025-12-31T23:59:59Z",
    "is_active": true,
    "created_at": "2024-01-01T00:00:00Z"
  }
}

用户ID已存在 (409)：

{
  "success": false,
  "message": "用户ID已存在"
}

邮箱已被注册 (409)：

{
  "success": false,
  "message": "邮箱已被注册"
}
6. 设置用户密码 需要 password 权限

修改用户密码，适用于密码重置、找回密码等场景。

POST /api/external/password
Content-Type: application/json
X-API-Key: your-token

{
  "user_id": 1,
  "password": "newpassword123"  // 新密码，至少6个字符
}

成功响应 (200)：

{
  "success": true,
  "message": "密码设置成功",
  "data": {
    "user_id": 1
  }
}
通用错误响应

未授权 (401) - 无效的API令牌：

{
  "success": false,
  "message": "未授权"
}

无权限 (403) - 令牌缺少所需权限：

{
  "success": false,
  "message": "无权限执行此操作"
}

用户不存在 (404)：

{
  "success": false,
  "message": "用户不存在"
}
